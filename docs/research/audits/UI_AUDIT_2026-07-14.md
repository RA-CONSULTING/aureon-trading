# Aureon UI Audit — 2026-07-14

A three-front audit of the `frontend/` unified UI (shell + build, components + data
honesty, the backend API surface it consumes), followed by a scoped **honesty pass**
and a new **two-tier validation harness**. This document records what was found, what
was fixed, and what is deliberately staged.

---

## 1. Architecture — healthy

The `src/shell/` layer is sound and is the template the rest of the UI should converge on:

- **Registry-driven** — `src/shell/nav.ts` is the single source of truth; the sidebar,
  router (`routes.tsx`), breadcrumbs, and command palette all derive from it. 24 routes
  across 6 sections. Adding a surface is one entry.
- **Lazy-loaded** — every route is `React.lazy`, so each dashboard is its own chunk.
- **Per-route error boundary** (`src/shell/Page.tsx::RouteErrorBoundary`), re-keyed on
  path in `ShellLayout.tsx` — a crashing dashboard shows a retry card, never takes down
  the shell.
- **Honest shell pages** — Overview / Connections / Providers / Operator Chat / Billing
  fetch real `/api/*` and degrade to explicit "gateway offline" states.

## 3. Honesty pass — what changed (this pass)

Nothing fabricated is shown as live; real panels are untouched; empty states stay honest.

- **`WarRoomDashboard.tsx`** — removed `mockSymbolStats` / `mockExchangeStatuses` / `mockKill`.
  Duck-commando intel now derives from **real** per-exchange connection state
  (`useMultiExchangeBalances`); the sniper leaderboard receives `[]` and shows an honest
  "No sniper telemetry yet — awaiting live trades" empty state. Header subline changed from
  "Real Quantum Data" to "Live Quantum State" to match verified provenance.
- **`warroom/DuckCommandoIntel.tsx`** — kill/PnL/positions fields made optional; the
  live-stats block only renders when a real telemetry source supplies them (never
  zero-filled or fabricated).
- **`warroom/SniperLeaderboard.tsx`** — added the empty state; header row hidden when empty.
- **`SchumannDataLoaders.tsx`** — `loadZipFrames` no longer fabricates (returns `[]` + an
  honest "ZIP import not implemented" warning); `csvToFrames` keeps real CSV frequencies but
  leaves `tensorField` empty instead of inventing random tensors.

## 4. Math.random triage

Every flagged analytics component was checked for reachability. **All are orphaned** — not
imported anywhere and not rendered as JSX in the legacy `App.tsx`, so none currently present
fabricated data to a user:

| Component | Verdict |
|---|---|
| `QuantumAnalytics`, `SchumannAnalytics` | Orphaned. 1 Hz `Math.random()` drift simulators (no real feed). Not routed. **Recommend wire-to-real-or-remove.** |
| `QuantumTradingConsole`, `QuantumDashboard`, `UnifiedTandemCore`, `TradingConsole` | Orphaned. Not imported anywhere. **Recommend remove or gate behind a labelled demo.** |
| `services/aureonService.ts` (`runAureonSimulation`) | A `Math.random()` simulator; only importer (`TradingConsole`) is itself orphaned. Fine as a *simulator*, but not reachable. |
| `cinema/*`, `Quantum*Mesh`, particle visualizers | `Math.random()` is **cosmetic** (animation) — legitimate, no change. |

Because no reachable live surface presents fabricated data after the §3 fixes, no
"simulated" badge component was added (adding UI to dead code would be its own cruft). The
recommendation for any future mounting of the orphaned simulators is to label them clearly
or wire them to a real source.

## 5. Validation harness — two tiers (mirrors `PRODUCTION_GRADE.md`)

**Strict tier (blocks CI) — the shell is the product surface:**
- `npm run build` — production bundle must compile.
- `npm run lint:shell` — `eslint src/shell --max-warnings 0` with `no-explicit-any`,
  `no-unused-vars`, `prefer-const`, `no-empty` as **errors** (via a `src/shell/**` override in
  `eslint.config.js`). ESLint reports only linted files, so this stays scoped despite the
  shell lazy-importing the legacy tree.
- Shell smoke E2E — `tests/shell-smoke.spec.ts` (Playwright): every representative route
  mounts without hitting the error boundary, the sidebar renders, and the body never scrolls
  horizontally. Backend-independent (asserts honest-offline, not live data).

**Foundation tier (informational ratchet — non-blocking):**
- `npm run typecheck` (`tsc -p tsconfig.app.json --noEmit`) — **baseline 17 errors** in the
  legacy tree (was 19; `lib` bumped ES2020→ES2021 cleared 2). Tracked to drive down; not
  gated, so no mass one-pass rewrite is forced.
- `npm run lint` (`eslint .`) — the disabled rules are now **warnings** across the ~269
  legacy components, so the tree can only improve.

**CI:** `.github/workflows/frontend-ci.yml` — `strict-tier` job (build · lint:shell · smoke)
blocks; `ratchet` job (typecheck + eslint counts) is `continue-on-error`.
`vite.config.ts` gained `chunkSizeWarningLimit: 800` so real bundle regressions stand out.

## 6. Staged (not done this pass — honest record)

- **Backend robustness for the UI** — apply `_json_safe` to `/api/status` + `/api/organism`
  (and ideally all JSON), guard the SSE generators, and reconcile the frontend-sends-no-auth
  gap.
- **Deploy correctness** — replace hardcoded `localhost`/`ws://localhost:8787` bases with
  proxy-relative / `VITE_*` config.
- **Consistency** — shared `apiFetch` + standardized `undefined|null|data` tri-state; single
  toaster; real 404 page.
- **`App.tsx` decomposition** — the 9,485-line legacy monolith behind the Legacy Console route.
- **Orphaned simulators** — wire-to-real or remove (§4).
