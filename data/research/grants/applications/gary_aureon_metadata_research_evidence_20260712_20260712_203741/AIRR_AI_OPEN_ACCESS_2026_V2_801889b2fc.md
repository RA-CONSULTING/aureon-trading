## Proposed Project

Title: Aureon Evidence OS for AI-Driven Scientific Discovery

Requested resource: 500,000 GPU hours on Isambard-AI

Duration: 12 months

Plain-language summary:

R&A Consulting and the Aureon Institute request compute access to develop and test a local-first AI evidence operating system for scientific discovery workflows. The work will use the existing open-source Aureon operator repository and the Harmonic Nexus Core research corpus to build reproducible AI workflow experiments: coherence-scored agent routing, evidence-pack generation, public research corpus indexing, and benchmarked multi-agent reasoning against public datasets and synthetic validation tasks.

This V2 application deliberately avoids unverifiable claims. The project does not claim validated consciousness, energy extraction, trading-profit, or defence capability. It frames HNC as a testable coherence and evidence-routing hypothesis for AI workflow design.

## Assessment Response Drafts

### 1. Alignment and Research Outcomes

The project will use advanced AI compute to build reproducible tools for scientific discovery workflows. The outcome is not a black-box assistant: it is an auditable operating layer that classifies sources, separates verified evidence from hypothesis, routes tasks to specialist agents, and writes proof artifacts before any external action. This aligns with AI-driven scientific discovery because it targets the process by which AI systems gather, classify, validate, and reproduce research evidence.

Expected outcomes:

1. Open-source workflow code and reproducible run instructions.
2. A public technical report describing the benchmarks and failure cases.
3. A source-indexed research corpus using claim classes A to D.
4. A demo of evidence-pack generation for grant and research workflows.
5. A follow-on bid package for Invest NI, InterTradeIreland, or academic validation.

### 2. Ambition and Novelty

The novel contribution is a coherence-scored evidence operating system for AI workflows. Instead of asking a model to produce an answer directly, Aureon builds a local evidence route: source inventory, claim classification, risk tagging, validation gate, and human-review status. HNC is treated as an algorithmic design hypothesis: whether golden-ratio-inspired temporal or spectral windows improve coherence scoring, task routing, or benchmark stability compared with conventional fixed windows.

This is ambitious because it combines open-source software engineering, AI workflow control, reproducible metadata, public research corpora, and auditable grant operations in a single local-first system.

### 3. Why AIRR Compute Is Required

The local machine can run the operator repo and small experiments, but it cannot perform large-scale benchmark sweeps, multi-agent simulation, or repeated model training across evidence-routing strategies. AIRR compute is needed for:

1. Parallel parameter sweeps across temporal windowing and coherence scoring methods.
2. Multi-agent simulation at thousands of runs per scenario.
3. Dataset preprocessing and model training at scales not possible on local hardware.
4. Robust ablation studies so HNC-inspired methods are compared against conventional baselines.
5. Reproducibility checks across seeds, data splits, and model sizes.

### 4. Responsible Research and Innovation

The project will not run live trading, make financial decisions, process private health data, or execute external submissions during the compute experiments. Data will be public, synthetic, or locally owned. Human review will remain required for grant submissions, legal filings, emails, and any external action.

Risk controls:

1. Data licence check before ingestion.
2. No private Gmail, Drive, or personal data in public datasets.
3. Source claim classes to prevent speculative language being presented as validated evidence.
4. Open methods and negative results included.
5. Human approval before any external publication or submission.
